# WATCH命令的实现
WATCH命令是一个乐观锁,它可以在EXEC命令执行之前，监视任意数量的数据库键，并在EXEC命令执行时，检查被监视的键是否至少有一个已经被修改过了如果是的话，服务器将拒绝执行事务，并向客户端防护代表事务执行失败的空回复。

Redis的事务和和传统的关系型数据库事务的最大区别在于，Redis不支持事务回滚机制，即使事务队列的某个命令在执行期间出现了错误，整个事务也会继续执行下去。
